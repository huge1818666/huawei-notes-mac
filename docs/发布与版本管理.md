# 发布与版本管理

本文档约定本项目的发布、打标、回退流程，方便后续同步到 GitHub 和做版本管理。

## 版本规则

- 仓库版本号以根目录的 `VERSION` 为唯一来源，例如 `1.0.0`。
- 应用构建时会自动把 `VERSION` 写入 `.app` 的 `CFBundleShortVersionString` 和 `CFBundleVersion`。
- 每次准备发布时，同时更新：
  - `VERSION`
  - `CHANGELOG.md`
  - `README.md` 中涉及版本的说明（如有）

## 建议发布流程

1. 确认工作区干净：

   ```bash
   git status
   ```

2. 更新版本号与变更记录：

   ```bash
   vim VERSION
   vim CHANGELOG.md
   ```

3. 构建应用并自测：

   ```bash
   ./build-app.sh
   open "build/华为备忘录.app"
   ```

4. 提交版本：

   ```bash
   git add .
   git commit -m "Release v$(cat VERSION)"
   ```

5. 创建版本标签：

   ```bash
   git tag -a "v$(cat VERSION)" -m "Release v$(cat VERSION)"
   ```

6. 推送分支和标签：

   ```bash
   git push origin main
   git push origin "v$(cat VERSION)"
   ```

7. 到 GitHub 创建 Release，并上传构建好的 `.app` 或压缩包。

## 回退建议

### 查看某个历史版本

```bash
git switch --detach v1.0.0
```

### 基于某个历史版本拉一条修复分支

```bash
git switch -c rollback/v1.0.0 v1.0.0
```

### 回退某次错误提交但保留历史

```bash
git revert <commit_sha>
```

不建议直接用破坏性命令覆盖公共历史；已经推送到 GitHub 的发布版本，优先用 `git revert` 修复，再重新发一个新版本。

## GitHub Release 建议

- Git Tag：`v1.0.0`
- Release Title：`华为备忘录 v1.0.0`
- Release Notes：优先从 `CHANGELOG.md` 摘要整理
- 附件：`华为备忘录.app` 或 `华为备忘录-v1.0.0-macos.zip`
